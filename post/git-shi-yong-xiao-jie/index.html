Git使用小结

发布于 2022-10-08

下面列举一些常用的语句，方便查询。
创建git仓库

git init

创建用户信息

git config --global user.email/name "myemail/myname"

查询git配置

cat ~/.gitconfig

查看文件状态

git status -s

添加索引

git add doc.txt

提交快照,-m""内添加说明，方便管理

git commit -m"V1"

从索引中删除

git rm doc.txt

上传,一般远程仓库名默认为origin

git push origin master

与远程仓库同步

git pull origin master

从远程仓库克隆

git clone Https/SSH

查看历史记录

git reflog

查看提交日志，从最近到最远

git log

版本回溯

# 返回上一个提交状态，上上一个就是HEAD^^
git reset --hard HEAD^
# 跳到指定位置
git reset --hard HEAD@{X}
or
git reset --hard 地址

git小结

什么是git仓库?

git仓库分为远程仓库和本地仓库，远程仓库是为了多人共享，合作的云数据库；而本地数据库是在个人电脑上建立的数据库，方便个人使用。
远程数据库和本地数据库的关系如图所示，每个人可以在个人电脑上编辑内容，然后上传到远程数据库。还可以从远程数据库获取其他人修改的内容，下载到本地数据库。

如何使用git进行版本管理?

git使用流程可以简述为：创建本地git仓库-->编辑内容-->添加索引-->提交快照-->建立远程git仓库-->上传/下载，具体流程参考猴子都懂的github入门：https://backlog.com/git-tutorial/cn/。
文件状态标记
Git有三种状态：已修改（modified）、已暂存（staged）和已提交（committed）。

  • 已修改：表示修改了文件，但还没保存到数据库
  • 已暂存：表示对一个已经修改的文件当前版本做了标记，使之包含在下次提交的快照中
  • 已提交：表示数据已经安全的保存在本地数据库中

状态标记符有：??（新添加的文件）、A（新添加进暂存区的文件）、M（修改过的文件）

  • 在本地数据库新建一个文件helloworld.txt，查看文件状态
  • 添加进入暂存区后，状态标记为A（在下一次提交快照前，所有新添加的文件都会标记为A）
  • 被修改的文件，根据M的位置有不同解释
  1. M前面有空格，说明文件在工作区被修改了，并且没有进暂存区。
  2. M在最左边，说明文件被修改了，并放进了暂存区。
  3. MM说明文件被修改放进暂存区后，又被修改了，且还没放进暂存区。
